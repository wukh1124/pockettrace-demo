## 響應式開發規則 (Vue 3)

### 核心交互規則

在移動設備上，滑鼠懸停（hover）並不存在，因此所有依賴 `hover` 觸發的 UI 必須調整：

- **禁止僅靠 Hover 顯示關鍵功能**：所有操作按鈕（如：刪除、編輯、分享）在手機版（螢幕寬度 < 768px）必須直接顯示，或透過明顯的「更多」圖標選單觸發 。
- **點擊代替懸停**：若有懸停顯示 Tooltip 的需求，在手機端應改為「點擊顯示/再次點擊隱藏」，或改用長按觸發 。​
- **視覺反饋**：使用 `:active` 偽類代替 `:hover` 來提供點擊時的觸覺/視覺反饋。

### 響應式布局與組件

Vue 3 開發中應遵循的結構性標準，確保跨裝置的一致性：

| 範疇         | 手機版優化要求 (Mobile-Specific Rules)                                                                 |
| ------------ | ------------------------------------------------------------------------------------------------------ |
| **觸控目標** | 所有可點擊元件（按鈕、連結）的最小尺寸須為 44x44 px，且間距至少 8px                                    |
| **斷點標準** | 統一使用 Tailwind 或 CSS Variables 定義斷點：Mobile (<640px), Tablet (640-1024px), Desktop (>1024px)。 |
| **表單輸入** | 輸入框應設定 `font-size: 16px`（避免 iOS 自動縮放），並根據內容正確設定 `inputmode` (如 `numeric`)     |
| **導航模式** | 桌面端側邊欄在手機版應自動轉換為「漢堡選單 (Drawer)」或「底部導航欄 (Bottom Nav)」。                   |

### Vue 3 程式碼實現指南

AI 在生成 Vue 組件時應自動包含以下邏輯：

- **動態類別判斷**：使用 Vue 3 的 `useWindowSize` 或 `matchMedia` 監聽裝置狀態。
  javascript
  `// 規則：AI 應自動偵測是否為觸控設備 const isTouchDevice = window.matchMedia("(pointer: coarse)").matches;`
- **圖片懶加載**：手機版應預設開啟 `v-lazy` 或原生 `loading="lazy"`，並針對移動端請求壓縮後的 WebP 格式圖片。
- **防止滾動穿透**：當 Modal 或 Drawer 開啟時，必須自動鎖定背景滾動（使用 `overflow: hidden`）。

### 樣式規範 (CSS/Tailwind)

- **避免固定寬度**：嚴禁在組件中使用固定的 `px` 寬度（如 `width: 500px`），應改用百分比、`vw` 或 `max-width` 。​
- **安全區域適配**：針對 iPhone 全螢幕手機，必須包含 `env(safe-area-inset-bottom)` 來避免內容被底部橫條遮擋。
